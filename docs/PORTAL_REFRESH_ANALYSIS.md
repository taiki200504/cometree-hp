# ユニオンポータルサイト刷新：現状との差分・改変度まとめ

## 1. 刷新後の想定構成（整理）

- **ポータル全体**: 企業・行政への価値 = ポテンシャル人材となる学生・学生団体にすぐつながる
- **学生向けHP**
  - ユニメイト紹介ページ（遷移）
  - 学生団体連合紹介ページ（遷移）
  - 登録学生団体のイベント情報
  - ユニオンのイベント情報
  - 拠点情報
- **独立LP**: ユニメイトのLP / 学生団体連合のLP
- **登録学生個人**: 個人向け案件、イベント情報、Discordリンク
- **登録学生団体**: 自団体LP、団体向け案件、学生団体基礎知識、Discord（代表）
- **Discord**: 登録情報閲覧、コミュニケーション
- **企業向け**: 主体的な学生をプレイヤーとして抜擢 → 案件登録、Discordアクセス、イベント委託
- **有料会員企業**: 案件を学生に流す、Discordで学生と交流
- **補足**: アンバサダー以外も BEGOPEN・クロスワン拠点で育てる（イベント送客・就活人材・地域関係人口）

### 2.2 現状のデータ・連携

- **ニュース**: Supabase `news` または Notion。公開APIあり。管理は `/admin/news`。
- **イベント**: Supabase `events` または Notion。管理は `/admin/events`。**公開の「イベント一覧」専用ページは現状なし**（コミュニティ・ポータル内で言及はあり）。
- **掲示板**: Supabase `board_posts`。一覧・詳細・お気に入りあり。
- **加盟団体・提携企業**: 管理画面（organizations, partners）あり。コミュニティ頁で一覧表示。
- **ユニメイト**: 「UNION Match」として **外部LP**（`https://union-match-lp.vercel.app/`）へのリンクのみ。**同一サイト内の「ユニメイト紹介ページ」はなし**。
- **学生団体連合紹介**: `/about` が実質該当。**単体の「学生団体連合LP」はなし**。
- **コミュニケーション**: **Slack** のみ。Discord の言及・リンクなし。
- **案件**: 学生は「インターン紹介」（Slack #intern-channel 案内）。企業は協賛・共創・相談窓口。**案件の「登録」や「学生へ流す」機能はHP上にはなし**。
- **拠点**: about に「17都道府県に活動拠点」の記載のみ。**BEGOPEN・クロスワン・拠点一覧ページはなし**。
- **有料会員企業**: イベント協賛等の料金表は `/services` にあるが、「案件を流す」「Discordで交流」という整理はなし。

## 3. 改変度の評価（テキストベース）

### 3.1 現状HPの「改変」で足りる部分（流用・修正）

- **学生向け入口の整理**
  - `/` または `/services?tab=student` を「学生向けHP」の入口として利用可能。文言・構成を「ユニメイト紹介・学生団体連合紹介・イベント・拠点」に合わせて**改変**。
- **学生団体連合紹介**
  - `/about` をベースに、学生団体連合に特化した**紹介ページ**として整理・追記。必要なら `/about/union` や `/student-union` などに分割し、LP化も検討可能（同一HP内で対応可能）。
- **企業向けページ**
  - `/services?tab=corporate` と `/join/corporate` がすでにある。ここに「主体的な学生をプレイヤーとして抜擢」「案件登録・Discordアクセス・イベント委託」「有料会員の特典（案件を流す・Discordで交流）」を**追記・再構成**すればよい。**新規ドメインや別HPは必須ではない**。
- **拠点情報**
  - 新規ページ 1 本追加（例: `/locations` または `/about/locations`）。BEGOPEN・クロスワン・その他拠点を一覧・マップで記載。**現状HPの拡張**で対応可能。
- **イベント情報の「見せる」**
  - 既存の `events` API・管理機能を利用し、**公開の「学生団体イベント一覧」「UNIONイベント一覧」ページ**を 1〜2 本追加（例: `/events`, `/community/events`）。データは既存のため**新規ページとUIの追加**が主な作業。
- **ニュース・掲示板・コミュニティ**
  - 既存の `/news`, `/board`, `/community` をそのまま活用しつつ、学生向け・企業向けのナビに**誘導を整理**する程度でよい。

→ これらは **「現状のこのHPを改変する」** で対応可能。別ドメイン・別サイトは不要。

---

### 3.2 現状HPの「追加」が必要な部分（新規ページ・新機能）

- **ユニメイト紹介ページ（同一サイト内）**
  - 現状は外部LPへのリンクのみ。刷新案では「学生向けHP → ユニメイト紹介ページ遷移」とあるため、**同一HP内に「ユニメイト紹介」ページを 1 本追加**するか、既存のサービス説明を「ユニメイト」として再ラベルし、必要なら詳細を `/about/unimate` などにまとめる必要あり。
- **ユニメイトのLP**
  - すでに `union-match-lp.vercel.app` が存在する場合は、**HPとは別のLPのまま**でよい。その場合、現状HP側の改変は「誘導・説明文の統一」と「紹介ページからのリンク」程度。LPをこのリポジトリに統合する場合は、`/unimate` や `/lp/unimate` のようなルートを新設する形になる。
- **学生団体連合のLP**
  - 現在は `/about` が紹介に相当。**独立したLP** が必要なら、例: `/lp/student-union` を新規作成し、縦長1ページで学生団体連合の価値・加盟メリットをまとめる。**新規ページ 1 本**で対応可能（同一HP内）。
- **登録学生・登録団体向け「得られること」の明示**
  - 現状、**「登録」に相当するフロー**（会員登録・ログイン）は auth と portal（加盟団体向け）に偏っている。  
  - 「登録している学生個人が得られる」「登録している学生団体が得られる」を**一覧で見せるページ**が必要なら、例: `/for-students/benefits`, `/for-organizations/benefits` を新規追加し、案件・イベント・Discord・自団体LP・学生団体基礎知識など**文言で整理**する。ログイン後のダッシュボードを将来作るなら、その入口もここに置ける。
- **Discord への移行・併用**
  - 文言・リンクを **Slack → Discord** に差し替え、または「Slack と Discord の役割」を説明する記載を追加。**コード上のリンク・文言の差し替えと、説明ブロックの追加**で対応可能。**新規サイトは不要**。
- **企業の案件登録・有料会員の整理**
  - 「案件情報の登録」「案件を学生に流す」は、現状HPには**フォーム・バックエンドが存在しない**。  
  - 案: お問い合わせフォームの一種として「案件登録」フォームを追加する／管理画面側で案件を登録し、学生向けに「案件一覧」を出す場合は、**案件用のデータ構造と公開API・一覧ページの新規実装**が必要。  
  - 有料会員企業向けの「案件を流す」「Discordで交流」は、**ページ上の説明と、既存の contact や加盟申請への誘導**で一度は賄い、実機能（案件投稿・マッチング）は別フェーズで検討可能。

→ これらは **「現状HP内の新規ページ・新機能」** で賄える。別サイトを増やさずに済む。

---

### 3.3 別途「HPとは別」で検討し得るもの

- **ユニメイトのLP**
  - 既存の `union-match-lp.vercel.app` をそのまま「ユニメイトのLP」として使い、このHPは「紹介＋リンク」に徹する場合は、**別サイトのまま**でよい。
- **学生団体ごとの「自団体LP」**
  - 「登録している学生団体が得られる → 自団体のLP」は、団体ごとに異なるURL・デザインが必要になり得る。現行の **community/organizations や団体詳細** を「簡易LP」として拡張するか、サブドメイン／別アプリで「団体ごとLP」を用意するかは、要件次第。**現状HPだけでは不足する可能性**があるが、まずは「団体詳細ページをLP的に強化する」で対応することも可能。
- **Discord 側の仕組み**
  - 「登録情報の閲覧」「コミュニケーション」は Discord サーバー・Bot・役割設計に依存するため、**HPの改変というより別途設計**。HPは「Discordに加入すると〇〇ができる」という説明とリンクで足りる。
- **案件マッチング・決済**
  - 案件の掲載・応募・有料会員の決済などは、**既存HPの範囲を超える**ため、別サービス・別システムとして検討する想定でよい。

## 4. 結論（改変量・別サイトの要否）

| 観点 | 結論 |
|------|------|
| **現状HPの改変量** | **中〜大**。トップ・サービス・参加方法・企業向け・About の**文言・構成・ナビの見直し**と、**新規ページの追加**（拠点、イベント一覧、ユニメイト紹介、学生団体連合LP、登録者向けメリットなど）が必要。 |
| **別HP・別サイトは必要か** | **基本は不要**。ユニメイトのLPを既存の外部LPに寄せるか、このHP内に統合するかは方針次第。学生団体連合のLP・企業向け・学生向け・拠点・イベントは**すべて現状の 1 サイト内に追加・改変**で対応可能。 |
| **新規に作るべきページ（案）** | ・学生向けメイン／学生団体連合紹介（またはLP）  
 ・ユニメイト紹介（またはLPをこのHPに統合）  
 ・拠点情報  
 ・イベント一覧（学生団体・UNION）  
 ・登録学生／登録団体のメリット説明  
 ・必要に応じて企業向け「案件登録」導線・案件一覧 |
| **新規に作るべき機能（案）** | ・イベント一覧の公開表示  
 ・（必要なら）案件登録フォーム／案件一覧表示  
 ・Discord 説明・リンクへの切り替え |

**テキストベースのまとめは以上です。** 実際の実装順序は「トップとナビの整理 → 学生向け／企業向けの文言統一 → 拠点・イベント一覧 → LP・紹介ページ」から進めると整理しやすいです。
